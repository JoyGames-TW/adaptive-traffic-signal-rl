Slide 11. Limitation of Adaptive Signal Control

中文標題：
自適應號誌控制的限制

主要內容：
- 運作機制：根據即時車流資料，動態調整號誌週期、綠燈秒數與相位順序。
- 優點：在台灣已展現縮短旅行時間的實務效益。
- 缺點：多數仍依賴交通工程師設計的啟發式最佳化規則。
- 面對高度非線性、多路口交互影響的複雜交通狀態時，適應能力仍受限。

Key Message：
傳統自適應系統的極限在於人為設計規則的邊界。

Slide 15. Markov Decision Process (MDP) Formulation

中文標題：
Markov Decision Process 建模

主要內容：
- 要讓 AI 學習控制紅綠燈，必須先將都市交通號誌控制建模為 Markov Decision Process。
- MDP 定義了智慧體如何在不確定環境中進行序列決策。
- 系統以 time-step t 推進，不斷在 S_t、A_t、R_t、S_{t+1} 之間循環。
- MDP 可表示為 <S, A, P, R, γ>。

Key Message：
MDP 是將實體交通問題轉換為數學最佳化問題的核心橋樑。

建議版面配置：
左側列出 MDP 數學表示法，右側對應交通情境中的 State、Action、Reward、Transition、Policy。

Slide 17. State Representation (S)

中文標題：
狀態表示設計

主要內容：
State Vector 代表路口的即時交通特徵，本研究狀態設計包含：
- 各車道排隊長度 Queue length
- 車輛平均等待時間 Waiting time
- 目前號誌相位 Current signal phase
- 剩餘綠燈時間 Remaining green time
- 車輛平均速度 Vehicle speed

設計理念：
提供 Agent 足夠的即時交通特徵，使其能理解目前車流狀態並預判擁塞趨勢。

Key Message：
精準的 State 設計能讓 Agent 看見車流的現在，並預測下一步交通壅塞風險。

Slide 18. Action Space Design (A)

中文標題：
動作空間設計

主要內容：
Action Space 定義 Agent 可以對號誌進行的控制指令。

本專案初期採用 Discrete Action Space：
- 選擇下一個號誌相位 Next signal phase
- 延長當前綠燈 Extend current green phase
- 切換至南北向綠燈
- 切換至東西向綠燈

採用離散動作的原因：
- 易於訓練與收斂
- 適合 DQN 與 PPO 的初步實驗
- 可快速驗證演算法有效性

Key Message：
離散動作空間能有效降低運算複雜度，同時滿足多相位動態切換需求。

建議版面配置：
展示四相位路口圖示，將 Action 表示為 a ∈ {0,1,2,3}。

建議視覺元素：
四相位號誌圖、Action selection 圖示。

Slide 19. Reward Function Design (R)

中文標題：
獎勵函數設計

主要內容：
Reward 是引導 Agent 行為的最重要指標。

若僅看通過車輛數，Agent 可能偏袒主幹道並犧牲支道公平性。因此，本研究採用 Joint Reward Function，最小化以下負向指標：
- 平均等待時間 Average waiting time
- 排隊長度 Queue length
- 路口延滯 Travel delay
- 環境碳排 Emissions

範例公式：
R_t = - (w_1 × Queue + w_2 × Delay + w_3 × Emission)

Key Message：
綜合獎勵函數確保整體路網的效率、公平性與環境永續。

建議版面配置：
中央放 Reward 公式，下方用四個 icon 分別代表 waiting、queue、delay、emissions。

建議視覺元素：
Reward formula、權重圖示、效率與公平性的平衡圖。

Key Message：
